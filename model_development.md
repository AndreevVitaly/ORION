# Собственная landmark-модель

## Целевой контракт

Модель получает RGB-изображение фиксированного размера и возвращает массив:

```text
[batch=1, vertex_count, 3]
```

Координаты `x` и `y` нормализованы относительно изображения. `z`
нормализуется относительно ширины изображения. Метаданные модели находятся в
sidecar-файле формата `model_contract.example.json`.

## Датасет

Каждый кадр должен иметь запись по `dataset_annotation_schema.json`.
Черновая предразметка создается функцией
`portrait_core.dataset.build_draft_annotation`, после чего обязательно
проверяется человеком.

Обязательные требования:

- документированное согласие на обработку биометрии;
- несколько кадров одного человека для проверки повторяемости;
- разделение train/validation/test по людям, а не по фотографиям;
- разнообразие камер, света, возраста и внешних особенностей;
- анфас как основной режим и контролируемые отклонения позы;
- ручная проверка автоматической предразметки;
- закрытый тестовый набор, не используемый при подборе модели.

MediaPipe и другие модели можно применять для предразметки, но не считать
эталоном без проверки человеком.

## Метрики

- normalized mean error по ориентирам;
- доля неудачных детекций;
- ошибка roll/yaw/pitch на размеченной подвыборке;
- повторяемость признаков по сериям одного человека;
- метрики отдельно по условиям съемки и группам данных;
- калибровка confidence.

## Этапы

1. Зафиксировать собственную топологию и руководство разметчика.
2. Собрать пилотный набор и провести двойную разметку части кадров.
3. Обучить baseline с переносом обучения.
4. Экспортировать модель в ONNX.
5. Подключить через `OnnxMeshAdapter`.
6. Сравнить с текущим backend на закрытом наборе.
7. Переключать production только после достижения заданных порогов.

Проект не использует геометрию лица для вывода характера, интеллекта,
психического состояния или поведения.
