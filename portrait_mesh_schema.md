# Portrait Mesh Schema v1

`portrait-mesh/1` — внутренний формат плотной сетки лица. Он отделяет
измерительное ядро от MediaPipe, ручной разметки и будущей собственной модели.

## Структура

```json
{
  "schema": "portrait-mesh",
  "schema_version": 1,
  "coordinate_system": "image-pixels",
  "dimensions": 3,
  "image_size": {"width": 1920, "height": 1080},
  "source": {
    "adapter": "mediapipe-face-landmarker",
    "topology": "mediapipe-478"
  },
  "vertices": [[100.0, 120.0, -4.0]],
  "semantic_map": {"nose_tip": 1},
  "metadata": {}
}
```

## Инварианты

1. Координаты `x` и `y` относятся к исходному изображению, а не к crop.
2. Порядок вершин определяется исходной топологией и не используется напрямую
   измерительным ядром.
3. `semantic_map` связывает проектные названия ориентиров с вершинами любой
   топологии.
4. Отсутствующая реальная глубина записывается как `0.0`; это не метрическая
   3D-координата.
5. Каждый отчет хранит версию схемы, название адаптера и исходную топологию.
6. Изменение смысла полей требует новой версии схемы.

## Граница зависимости

MediaPipe разрешено использовать только внутри
`portrait_core/adapters/mediapipe_adapter.py`. Остальное приложение работает
с `portrait-mesh/1` и именованными проектными ориентирами.

Новая модель должна реализовать `FacePointAdapter.extract_mesh()` и вернуть
тот же контракт. Ее количество и порядок вершин могут отличаться.

Для точных зон модель может передать в `metadata.contours` именованные
последовательности индексов: `face_oval`, `jaw`, `left_eye`, `right_eye`,
`nose`, `lips_outer`, `mouth_opening`, `upper_lip`, `lower_lip` и брови.
Если контур отсутствует, ядро использует семантический fallback.

## Следующие версии

Планируемые расширения:

- уверенность и видимость каждой вершины;
- связи контуров и треугольная топология;
- маски анатомических зон;
- сведения о калибровке камеры;
- доверительные интервалы вычисленных признаков.

Канонические координаты и оценка позы находятся в отчете отдельно от исходной
сетки. Это сохраняет оригинальный результат backend без необратимых изменений.
